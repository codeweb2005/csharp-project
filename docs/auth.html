Vendor Security

Two-Layer Security Model

Layer 1 — Frontend (UX only)

  • useCurrentUser() decode JWT từ localStorage
  • isAdmin / isVendor flags → ẩn/hiện UI elements
  • Sidebar items có adminOnly: true → ẩn với Vendor
  • DashboardRoute → Admin Dashboard vs Vendor Dashboard
  • Chỉ là UX — có thể bypass!

Layer 2 — Backend (Authoritative)

  • [Authorize(Roles = "Admin")] — Admin only routes
  • [Authorize(Roles = "Admin,Vendor")] — shared routes
  • [AllowAnonymous] — mobile endpoints (nearby, public, audio stream)
  • Vendor write: if (poi.VendorUserId != callerId) return FORBIDDEN
  • Vendor read: controller lấy vendorPoiIds từ DB fresh, truyền vào service → WHERE POIId IN (...)

Refresh Token Security

  • 64-byte cryptographically secure random
  • Lưu DB dưới dạng SHA-256 hash (không phải plaintext)
  • Rotation: mỗi lần dùng → tạo cặp mới, revoke cũ
  • Reuse detection: token đã revoked được dùng → detect → revoke toàn bộ sessions của user đó
  • TTL: Jwt:RefreshExpiryDays (default 7 ngày)

Frontend Token Storage

  • localStorage.setItem('accessToken', jwt)
  • localStorage.setItem('refreshToken', token)
  • localStorage.setItem('user', JSON.stringify({...}))
  • 401 interceptor trong api.js → request() → tự động refresh → retry
  • Refresh fail → clearTokens() → redirect /login
KHÔNG bao giờ xóa hoặc làm yếu [Authorize] attributes. Không đổi [Authorize(Roles="Admin")] thành [AllowAnonymous]. Không thêm [AllowAnonymous] vào admin/vendor endpoints. Xem CLAUDE.md § 5.
