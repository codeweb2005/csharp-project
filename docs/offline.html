Offline & Sync Strategy — MAUI

App Launch → Load SQLite Cache (offline) → Online check → GET /pois/nearby (full) or GET /sync/delta (since=...) → Upsert SQLite

Initial Load (First launch)

  • GET /api/v1/pois/nearby?lat&lng&radius=2000&langId=1
  • Lấy đầy đủ POIs + translations + audioUrls trong vòng 2000m
  • Upsert toàn bộ vào SQLite
  • Lưu timestamp lastSyncedAt

Delta Sync (Subsequent launches)

  • GET /api/v1/sync/delta?since=2026-03-01T00:00Z&languageId=1
  • Auth: Customer JWT
  • Backend: SELECT WHERE UpdatedAt > since
  • Return: {modified: [...], deleted: [...]}
  • Mobile: upsert modified + soft-delete deleted records

Visit Upload (POST /sync/visits)

  • Auth: Customer or Anonymous
  • Queue visits locally khi offline → upload batch khi có mạng
  • Batch request: {visits: [{poiId, triggerType, narrationPlayed, listenDuration, visitedAt}]}
  • Backend: INSERT batch + TotalVisits += 1 per POI

Audio Cache (SQLite)

  • NarrationPlayer lưu MP3 path vào SQLite sau khi stream
  • Lần sau: kiểm tra SQLite trước → play local (0ms)
  • Phase 3: OfflinePage.xaml — user tải toàn bộ audio ZIP

Offline Packages (Admin Only)

Admin tạo offline data packages cho mobile app tải xuống — ZIP chứa toàn bộ POI data + audio files của một ngôn ngữ/khu vực.

EndpointAuthMô tả
GET /api/v1/offlinepackagesAdmin onlyList offline package definitions
POST /api/v1/offlinepackagesAdmin onlyCreate package definition (name, languageId, area, ...)
POST /api/v1/offlinepackages/{id}/buildAdmin onlyTrigger build — generate ZIP file async
GET /api/v1/offlinepackages/{id}/statusAdmin onlyCheck build status (pending / building / completed / failed)
GET /api/v1/offlinepackages/{id}/downloadCustomer ✅Download ZIP file — dùng bởi mobile OfflinePage
OfflinePage.xaml (Phase 3): User xem danh sách available packages, tải về, theo dõi progress bar, và manage local storage. Sau khi tải, app dùng data từ ZIP — không cần network để xem POI và nghe audio.
