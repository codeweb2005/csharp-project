Frontend Panels

Hai Frontend Riêng Biệt

Admin Panel — port 5173

admin-frontend/. Shared cho cả Admin và Vendor nhưng với UI khác nhau theo role.

  • Admin thấy: tất cả POIs, Users, Settings, Offline Packages, full Analytics
  • Vendor thấy: chỉ POI của mình, menu, audio của mình, analytics riêng
  • DashboardRoute → render Dashboard (Admin) hoặc VendorDashboard (Vendor)
  • Sidebar: items có adminOnly: true → ẩn với Vendor

Vendor Panel — port 5174

vendor-frontend/. Codebase riêng biệt dành riêng cho Vendor.

  • Cấu trúc tương tự admin-frontend nhưng là 2 codebase độc lập
  • Chỉ có các tính năng Vendor cần
  • Cùng api.js pattern, cùng React Router v7, cùng Ant Design 6.3
UI Library (bắt buộc): Ant Design 6.3 (KHÔNG dùng Material UI, Chakra), Lucide React (KHÔNG Font Awesome), Recharts (KHÔNG Chart.js, D3), Leaflet (KHÔNG Google Maps SDK), Vanilla CSS (KHÔNG Tailwind).

Admin APIs

Admin-Only Features

User Management

  • GET /users — list với filter role, search
  • POST /users — Admin tạo Vendor/Admin accounts (đặt password tạm)
  • PUT /users/{id} — edit thông tin
  • PATCH /users/{id}/toggle — disable/enable user
  • POST /users/{id}/reset-password — reset → return generated password

System Settings

  • GET/PUT /settings — system configuration
  • PUT /settings/maintenance — toggle maintenance mode
  • POST /settings/generate-api-key — generate new API key

Categories

  • GET /categories — Anonymous (mobile uses this)
  • POST/PUT/DELETE /categories/{id} — Admin only
  • PATCH /categories/{id}/toggle — Admin only
  • Fields: icon (emoji), color (hex), translations per language

api.js — HTTP Client

  • TẤT CẢ API calls đi qua api.js → request()
  • Auto-refresh 401: đọc refreshToken → POST /auth/refresh → retry
  • Refresh fail → clearTokens() → redirect /login
  • KHÔNG bao giờ dùng fetch() trực tiếp trong components
